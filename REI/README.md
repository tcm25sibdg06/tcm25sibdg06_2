## Descrição do Trabalho

No quotidiano de estudantes e profissionais, a gestão do volume de atividades diárias é frequentemente realizada de forma descentralizada e analógica (através de anotações em papel, blocos de notas e em muitos casos, confiando exclusivamente na memória). Como consequência direta, os utilizadores enfrentam a perda frequente de prazos, stresse devido à sobrecarga mental, desorganização entre os âmbitos pessoal e profissional, e uma total falta de rastreabilidade sobre o seu progresso e produtividade histórica.
Para resolver este cenário, propõe-se o desenvolvimento de um Gerenciador de Tarefas, um Sistema de Informação baseado num modelo de dados relacional que automatiza o fluxo de trabalho, a Workity. A ferramenta servirá para registar, atualizar e monitorizar tarefas (com estados como Pendente ou Concluído), além de criar automaticamente espaços de trabalho (workspaces) inteligentes e segmentados assim que um novo tipo de atividade (como Escola ou Trabalho) for detetado pelo sistema.
A grande vantagem desta solução informática é a eliminação do esforço manual de triagem, oferecendo ao utilizador uma clareza visual que potencia o foco e reduz as distrações. Adicionalmente, a transição para uma base de dados normalizada garante a integridade da informação em tempo real, reduz o erro humano por esquecimento e otimiza o tempo, transformando o caos de dados dispersos num ambiente de trabalho altamente eficiente e seguro.

O sistema funciona como um organizador inteligente que ajuda o utilizador a controlar os seus afazeres através de regras bem definidas. Cada pessoa cria um perfil único com email e senha (guardada de forma segura), recebe um número de identificação único (ID) além da data em que criou a conta e a última vez que a atualizou e a partir daí pode registar várias tarefas.
 Cada tarefa tem o seu próprio ID definidas caracterizadas por uma descrição, o seu estado (se está "Pendente" ou "Concluído") associando-as a categorias predefinidas como escola ou trabalho. O sistema possui uma interface de lógica aplicacional que atua de forma automatizada: sempre que o utilizador cria uma tarefa com uma categoria nova, o próprio sistema deteta isso e cria automaticamente um espaço de trabalho (workspace) exclusivo para essa categoria. Este espaço recebe um nome descritivo e fica imediatamente vinculado à identidade daquele utilizador e ao tipo de tarefa que o originou, recebendo também um número de identificação único para que o computador nunca o confunda.
 A partir desse momento, todas as tarefas futuras desse mesmo tipo entram sozinhas para esse espaço. O objetivo desta base de dados é gerir a informação de forma a facilitar a vida do utilizador, separando o lado académico do profissional para que ele não se disperse e encontre tudo num só lugar.
